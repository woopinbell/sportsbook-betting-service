# 007 — 슬립 규칙 + slippage 경계 테스트 (`test(validation): cover slip rules and slippage boundaries`)

## 개요 — 왜 이 변경이 필요한가

005/006의 검증은 **경계**에서 정확해야 한다(at/over the limit, floor exactly). Redis를 mock해 빠르고 결정적인 단위로 경계를 박는다.

## 작업 순서

1. `BetSlipValidatorTest` — L1/L2 구분, L4 at/over, L5 over/under, single 면제, stake 경계.
2. `OddsSlippageCheckerTest` — floor exactly / 개선 / just-below / market 상태 / unpriced.

## 작업 내역

### 구조 규칙 경계

- **L1 vs L2 구분**: 같은 market을 공유하면 L1으로(L2 아님) 보고됨을 단언.
- **L4** at the limit / over the limit, **L5** 곱이 cap over/under.
- **single 면제**: single 슬립은 L1/L2를 받지 않음.
- **stake 경계**: per-currency min/max + 미지원 currency 거절.

### slippage 경계 — 정확한 floor

제출 odds 2.0000이면 3% floor = **1.9400**. 테스트가 박는 경계:
- floor에서 정확히(1.9400) → 통과,
- odds 개선 → 통과,
- floor 바로 아래(1.9399) → 거절,
- suspended / closed / missing-status market → 거절,
- unpriced selection → 거절.

Redis를 mock하므로 경계가 **정확**하고(실 Redis 타이밍 흔들림 없음) 테스트가 빠른 단위로 유지된다.

## 결과

005/006의 모든 분기와 경계가 단위로 박힘.

- Redis mock으로 결정적·빠름.
