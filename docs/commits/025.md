# 025 — BetVoided consume → VOIDED (`feat(settlement): consume BetVoided, flip an accepted bet to VOIDED`)

## 개요 — 왜 이 변경이 필요한가

경기가 취소/연기되거나 market/admin void가 적용되면 슬립 전체를 void하고 stake를 전액 환불해야 한다(ADR-0012). settlement-service가 `BetVoided`를 발행하면 betting이 consume해 ACCEPTED→VOIDED로 전이한다. **이건 SETTLED(result=VOID)와 다른 별도 terminal status**라는 게 핵심.

## 작업 순서

1. `V4__bet_void_reason.sql` — void_reason(check) 추가, resolved_at 재사용.
2. `Bet.voidBet(reason, now)` 전이 + betting-local `VoidReason` enum.
3. `SettlementResultListener.onBetVoided` — 두 번째 listener.

## 작업 내역

### VOIDED(전체 슬립) vs SettlementResult.VOID(per-selection)는 다르다

이 구분이 이 커밋의 핵심이다.
- **VOIDED**(이 커밋): 경기 취소/연기 등으로 **슬립 전체**가 void → wallet이 **전액 환불**. 별도 terminal status.
- **SETTLED + SettlementResult.VOID**(024): SETTLED인데 *한 selection*의 결과가 VOID인 settle-time 부분 환불.

둘을 한 상태로 뭉치면 환불 의미(전액 vs 부분)가 뒤섞인다. 그래서 별도 status로 모델링.

```sql
ALTER TABLE bet ADD COLUMN void_reason VARCHAR(24);
ALTER TABLE bet ADD CONSTRAINT bet_void_reason_valid
    CHECK (void_reason IS NULL OR void_reason IN ('EVENT_CANCELLED','EVENT_POSTPONED','MARKET_VOID','ADMIN_VOID'));
```

void는 V3의 `resolved_at`을 terminal 전이 시각으로 **재사용**한다(SETTLED/VOIDED 공통).

### Avro enum → betting-local enum

```java
VoidReason reason = VoidReason.valueOf(event.getReason().name());   // wire enum → 도메인 enum
service.applyVoided(betId, reason, event.getVoidedAt());
```

Avro의 void 사유 enum을 betting-local `VoidReason`으로 매핑해, **wire schema가 JPA aggregate에 절대 새지 않게** 한다(024와 동형). 환불 자체는 wallet-service 책임 — betting은 사유 기록 + 상태 전이만.

### 멱등 by betId

SETTLED 경로처럼 재배달된 `BetVoided`의 bet이 이미 VOIDED면 **no-op**. unknown/불법 전이는 throw → DLT.

## 결과

void된 베팅이 VOIDED로 전이(사유 기록). SETTLED(result=VOID)와 의미가 분리됨. wallet이 전액 환불.

## 요약

- **VOIDED(전체 슬립, 전액 환불) ≠ SETTLED+VOID(per-selection 부분 환불)** → 별도 terminal status.
- V4 void_reason(check) 추가, resolved_at 재사용. Avro enum → betting-local VoidReason(wire 비누설).
- 멱등 by betId(이미 VOIDED면 no-op), 환불은 wallet 책임.

## 기억/설명 Level

- **L3 (외워서 설명, 면접 단골)** — *VOIDED vs SettlementResult.VOID*: VOIDED는 경기 취소/연기로 슬립 전체 void + 전액 환불(별도 terminal status). SETTLED+VOID는 한 selection의 settle-time 부분 환불. 별도 상태라야 환불 의미가 안 섞인다. BetVoided도 betId 멱등(이미 VOIDED면 no-op), unknown/불법 전이는 DLT. Avro enum을 betting-local로 매핑(wire 비누설).
- **L2 (문서 보며 설명)** — V4 void_reason check, resolved_at 재사용, 환불은 wallet 책임이고 betting은 상태/사유만.
- **L1 (읽으면 됨)** — void 사유 4종, topic `bet.voided.v1`.
