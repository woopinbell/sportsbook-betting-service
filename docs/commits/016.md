# 016 — outbox Kafka publisher (스케줄 drain) (`feat(outbox): scheduled Kafka publisher for the outbox`)

## 개요 — 왜 이 변경이 필요한가

013이 outbox 행을 *쓰는* 쪽이라면, 이건 그 행을 Kafka로 *내보내는* 쪽이다. 스케줄 `OutboxPublisher`가 미발행 행을 일정 주기로 drain한다 — at-least-once + partition 순서 보존.

## 작업 순서

1. `KafkaConfig` — `(String, byte[])` producer.
2. `OutboxPublisher` — fixed-delay drain.
3. 백그라운드 잡(outbox + reconciliation) 스케줄 설정 + 테스트 프로파일 parking.

## 작업 내역

### tick마다 fetch → send → ack 대기 → stamp

```java
@Scheduled(fixedDelayString = "${betting.outbox.poll-interval-ms:1000}")
@Transactional
public void publishPending() {
  List<OutboxEvent> batch = repository.findUnpublished(PageRequest.of(0, BATCH_SIZE)); // 미발행 오래된 순
  for (OutboxEvent event : batch) {
    if (sendBlocking(event)) event.markPublished(clock.instant());  // ack 받으면 stamp
  }
}
```

각 tick이 bounded batch(100)를 fetch, 각 행의 Avro payload를 보내고 **ack를 기다린 뒤** `published_at`을 찍는다. ack에 실패한 행은 미발행으로 남아 **다음 tick에 재시도** — 이 "ack까지 재시도" 동작이 outbox를 **at-least-once**로 만든다. 발행된 행을 삭제하지 않으므로 downstream replay는 `published_at` non-null 행을 다시 보내는 것으로 충분(별도 archive 테이블보다 운영이 쌈).

### producer 배선 — byte[] + idempotent + acks=all

```yaml
# KafkaConfig → ByteArraySerializer, enable.idempotence: true, acks: all
```

payload가 raw Avro bytes라 `(String key, byte[] value)` producer다(Schema Registry 없음, ADR-0014). `acks=all` + idempotent producer로 브로커 측 중복·유실을 줄인다.

> **C/C++ 관점**: `@Scheduled fixedDelay`는 cron 같은 OS 타이머가 아니라 Spring의 스케줄러 스레드가 *이전 실행 종료 후* 고정 지연으로 다시 부르는 것이다. `KafkaTemplate.send(...).get(timeout)`은 비동기 send를 **블로킹**으로 바꿔 ack를 기다린다(Future.get).

### 테스트 프로파일에서 잡 parking

`application-test.yml`에서 outbox + reconciliation poll을 **24h로 주차(park)** 한다 — 테스트가 백그라운드 drain에 흔들리지 않고 잡을 *명시적으로* 구동(017/019). production 기본은 outbox 1s, reconciliation 10s.

## 결과

outbox 행이 Kafka로 안정 drain. at-least-once + partition 순서 보존. 중복은 소비자 멱등(013 partition key + 소비자 측)이 흡수.

## 요약

- 스케줄 drain: fetch(미발행 batch) → send → **ack 대기** → `published_at` stamp. 실패는 다음 tick 재시도 = at-least-once.
- producer: byte[](Avro raw) + idempotent + acks=all.
- 테스트 프로파일에서 잡 24h 주차 → 명시 구동.

## 기억/설명 Level

- **L3 (외워서 설명)** — *outbox publisher가 at-least-once인 이유*: 각 tick이 미발행 행을 보내고 ack를 기다린 뒤에만 published_at을 찍는다. ack 실패 행은 남아 다음 tick 재시도. 발행 행은 안 지워 replay 가능. 중복은 소비자 멱등이 흡수(exactly-once 아님, V1).
- **L2 (문서 보며 설명)** — `@Scheduled fixedDelay` 의미, `send().get(timeout)` 블로킹 ack, acks=all + idempotent producer, 테스트 주차.
- **L1 (읽으면 됨)** — BATCH_SIZE 100, poll 1s(prod)/24h(test).
