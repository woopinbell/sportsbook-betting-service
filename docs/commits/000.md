# 000 — 프로젝트 골격 (`chore(project): initialize betting-service layout`)

## 개요 — 왜 이 변경이 필요한가

betting-service는 Phase 3의 핵심 repo이자 시스템에서 가장 임팩트가 큰 서비스다. 본 구현 전, 다른 서비스와 **동일한 빌드/관측성/lint 컨벤션**으로 뼈대를 세운다. 코드 0줄짜리 기능 커밋이지만, 이 repo가 따를 스택과 규칙을 박는다.

## 작업 순서

1. `git init` (main 브랜치).
2. `pom.xml` — Java 17 + Spring Boot 3.2.x + Maven, shared-protocol(JitPack/mavenLocal) 의존.
3. `.gitignore`(**`CLAUDE.md` 라인 필수**), `README.md`(영문 박스 + 한국어 본문), `docs/commits`·`docs/reflection`에 `.gitkeep`.
4. 빈 패키지 `com.sportsbook.betting` + `BettingServiceApplication`.
5. logback(JSON/console 이중 프로파일), Checkstyle 룰셋, `application.yml`.
6. `mvn verify` 그린 + `git status`로 CLAUDE.md ignored 확인 후 커밋.

## 작업 내역

### 스택과 컨벤션을 형제 서비스에서 복사

상위 `sportsbook/CLAUDE.md`의 결정 사항(Java 17, Spring Boot 3.2.x, Maven, Spotless + Checkstyle, JUnit5 + AssertJ + Mockito + Testcontainers)을 그대로 반영했다. Spotless(google-java-format), 공유 Checkstyle, 이중 프로파일 logback, Actuator liveness/readiness + Prometheus, `@Tag("load")` surefire 제외까지 일관.

> **C/C++ 관점**: Maven `pom.xml`은 CMake처럼 빌드 그래프를 선언하지만, **BOM(Bill of Materials)** 이 다르다. `spring-boot-starter-parent`를 부모로 두면 수십 개 의존성의 버전이 한 곳에서 고정된다 — 헤더/라이브러리 버전을 손으로 맞추던 것과 대비. `mvn`/`mvnw`는 macOS zsh에서 CLI로 돌린다(IntelliJ 가정 없음).

### CLAUDE.md는 git에서 영구 제외

`.gitignore` 1~2번째 줄에 `CLAUDE.md`. 이건 AI 세션의 운영 컨텍스트 파일이지 프로젝트 산출물이 아니다(sportsbook/CLAUDE.md 정책). 포트폴리오 면접관에게는 README + ADR + docs/만 보인다. `git status`로 untracked + ignored 확인.

### README 영문 박스 (ADR-0016)

해외 면접관 첫 클릭 대응으로 상단에 영문 요약 박스(서비스 한 줄 설명 + 시스템 위치 + 핵심 스택), 본문은 한국어. 이 박스는 retrospective 마지막 `docs(readme)` 커밋에서 측정 숫자까지 확장된다.
