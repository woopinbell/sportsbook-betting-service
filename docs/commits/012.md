# 012 — V2 transactional outbox 스키마 (`build(flyway): add V2 transactional outbox schema`)

## 개요 — 왜 이 변경이 필요한가

수락된 베팅의 **사후** 이벤트(`BetPlacedRequested`)를 Kafka로 안정적으로 내보내려면 Transactional Outbox 패턴이 필요하다(ADR-0006). 이벤트를 수락과 **같은 트랜잭션**에서 DB에 쓰면, 이벤트가 수락과 절대 어긋날 수 없다. 이 커밋은 그 `outbox_event` 테이블을 만든다.

## 작업 순서

1. `V2__outbox.sql` — `outbox_event` 테이블 + unpublished 부분 인덱스.

## 작업 내역

### 왜 outbox인가 — dual-write 문제

"DB에 bet 저장 + Kafka에 이벤트 발행"을 따로 하면 둘 사이에 프로세스가 죽었을 때 **하나만 성공**한다(dual-write 불일치). outbox는 이벤트를 *DB 행*으로 같은 트랜잭션에 쓰고(원자적), 별도 publisher(016)가 그 행을 나중에 Kafka로 drain한다. 이벤트는 수락과 함께 커밋되거나 함께 롤백된다 — 절대 따로 놀지 않는다.

```sql
CREATE TABLE outbox_event (
    event_id      UUID PRIMARY KEY,
    topic         VARCHAR(64)  NOT NULL,
    partition_key VARCHAR(64)  NOT NULL,
    schema_name   VARCHAR(64)  NOT NULL,
    payload       BYTEA        NOT NULL,   -- Avro binary (no Schema Registry, ADR-0014)
    created_at    TIMESTAMP WITH TIME ZONE NOT NULL,
    published_at  TIMESTAMP WITH TIME ZONE          -- NULL = 미발행
);

CREATE INDEX ix_outbox_unpublished
    ON outbox_event (created_at) WHERE published_at IS NULL;   -- 부분 인덱스
```

### 부분 인덱스로 poll을 싸게

`WHERE published_at IS NULL` **부분 인덱스(partial index)** 가 핵심이다. publisher는 "미발행 행 오래된 순"만 계속 poll하는데, 발행된 행이 테이블 대부분을 차지해도 인덱스는 *미발행*(드문 상태)만 담아 스캔이 작게 유지된다.

> **C/C++ 관점**: 부분 인덱스는 PostgreSQL이 조건을 만족하는 행만 인덱스에 넣는 기능 — "hot한 소수 상태"에 인덱스를 좁혀 크기·유지비를 줄인다. payload가 `BYTEA`(raw bytes)인 건 Avro binary를 그대로 담기 때문(013) — Schema Registry 없이 소비자가 같은 생성 클래스를 pin(ADR-0014).

## 결과

outbox 테이블 + 효율적 poll 인덱스. 013이 write side, 016이 publisher.

## 요약

- Transactional Outbox(ADR-0006)로 dual-write 불일치를 막음 — 이벤트를 수락과 같은 트랜잭션에 DB 행으로.
- payload는 Avro binary `BYTEA`(no Schema Registry, ADR-0014).
- `WHERE published_at IS NULL` 부분 인덱스로 publisher poll을 싸게.

## 기억/설명 Level

- **L3 (외워서 설명)** — *왜 outbox인가*: DB 저장 + Kafka 발행을 따로 하면 dual-write 불일치(하나만 성공). outbox는 이벤트를 같은 트랜잭션에 DB 행으로 써 원자성 확보, publisher가 나중에 drain(at-least-once). 부분 인덱스로 미발행만 스캔.
- **L2 (문서 보며 설명)** — 부분 인덱스가 poll을 작게 유지, payload BYTEA = Avro binary(Schema Registry 없음, 소비자가 클래스 pin).
- **L1 (읽으면 됨)** — published_at NULL = 미발행, 컬럼 구성.
