# 009 — System 베팅 조합·payout 증명 (`test(domain): prove System bet combinations and payout`)

## 개요 — 왜 이 변경이 필요한가

008은 가장 error-prone하고 면접에서 가장 많이 묻는 경로다. 조합론과 돈 계산을 **알려진 값**으로 박아 회귀를 차단한다.

## 작업 순서

1. binomial 알려진 값 + 엣지.
2. combinations 정확 열거 + 불변(정렬/distinct/count=binomial).
3. lineCount/totalStake.
4. maxPayout 케이스(single/multiple/system/floor/currency).

## 작업 내역

### binomial / combinations

- 알려진 값: **C(15,7)=6435**, 엣지(k=0, k=n, k>n).
- `combinations(3,2)`를 정확히 열거(`[0,1],[0,2],[1,2]`), count가 binomial과 일치, 조합이 정렬·distinct, 최대 허용 C(15,7)이 실제로 열거됨.

### 돈 계산

- `lineCount` + `totalStake`를 Single/Multiple/System 각각 확인.
- **maxPayout**:
  - single = stake × odds,
  - multiple = stake × product,
  - **system 2-of-3** = 세 조합 곱의 합(예: 26×),
  - 2-of-4 동일 odds(24×),
  - **K=N**이 대응하는 multiple과 같음(C(N,N)=1 line),
  - sub-unit 분수가 **floor**됨,
  - currency 보존.

system 2-of-3을 손으로 검산할 수 있게 작은 수로 박는다 — C(3,2)=3 line, 각 line이 2개 odds 곱, 그 합 × unit.

## 결과

조합·payout이 알려진 값으로 고정. 008의 한 공식이 세 타입에서 정확함을 증명.

## 요약

- binomial(C(15,7)=6435 포함)·combinations 정확 열거·불변 검증.
- maxPayout을 single/multiple/system(2-of-3, 2-of-4, K=N)·floor·currency로 박음.
