# 023 — shared-protocol 0.2.0 bump (`build(deps): bump shared-protocol to 0.2.0-SNAPSHOT`)

## 개요 — 왜 이 변경이 필요한가

Phase 4(정산 consumer)의 시작. betting이 ACCEPTED 베팅을 SETTLED/VOIDED로 전이하려면 **정산 이벤트 family**가 필요한데, 그게 shared-protocol 0.2.0에 추가됐다. 이 커밋은 의존 버전만 올린다(코드 변경 없음).

## 작업 순서

1. `pom.xml`의 shared-protocol 버전 0.1.x → `0.2.0-SNAPSHOT`.

## 작업 내역

0.2.0이 추가하는 것: `BetSettled`, `BetVoided`, `SettlementResultAvro`, `VoidReason`(Avro). 이것들을 consume해 사후 SETTLED/VOIDED 전이를 구동한다(Phase 4, ADR-0006 비동기 정산). dev 중에는 mavenLocal에서 resolve(shared-protocol을 `mvn install` 후).

> **C/C++ 관점**: `-SNAPSHOT`은 Maven의 "아직 릴리스 안 된 개발 버전" 표식 — 같은 좌표라도 매번 최신을 가져온다(릴리스 버전은 불변). mavenLocal(`~/.m2/repository`)은 로컬 빌드 산출물 캐시라, shared-protocol을 로컬에서 빌드·install하면 betting이 그걸 본다. C++의 로컬 헤더/라이브러리 설치 경로와 비슷.

## 결과

정산 이벤트 클래스를 컴파일 타임에 사용 가능. 024~026이 이를 consume.

## 요약

- shared-protocol 0.2.0-SNAPSHOT bump — 정산 family(BetSettled/BetVoided/SettlementResultAvro/VoidReason) 도입.
- 코드 변경 없음(버전만), mavenLocal resolve.
