# 008 — SystemBetCalculator (K-of-N 조합 + payout) (`feat(domain): add SystemBetCalculator for K-of-N combinations + payout`)

## 개요 — 왜 이 변경이 필요한가

System(K-of-N) 베팅은 이 서비스에서 가장 error-prone하고 면접에서 가장 많이 묻는 계산이다. 슬립을 조합으로 전개해 **지갑 차감액(totalStake)** 과 **최악 시 지급액(maxPayout)** 을 계산한다(ADR-0008). 핵심 통찰: 세 슬립 타입이 **한 공식의 특수 케이스**다.

## 작업 순서

1. 슬립 → line(누적) 전개 모델 정의.
2. `lineCount` / `totalStake` / `maxPayout`.
3. `combinations`(k-subset 열거, pruning) + `binomial`(곱셈 공식).

## 작업 내역

### 한 공식: 슬립은 line 집합

`Bet.stake`를 **per-line(unit) stake**로 정의한다. 슬립은 line(accumulator) 집합이고, 각 line은 그 위의 모든 selection이 이기면 `unit × product(line의 odds)`를 지급한다.

| 타입 | line 수 |
|---|---|
| Single | 1 line (1 selection) |
| Multiple | 1 line (전체 N selection) |
| System(K-of-N) | C(N,K) line (K-subset마다 하나) |

따라서:
- **totalStake**(지갑 차감, ADR-0017) = `unit × lineCount`
- **maxPayout**(모든 selection이 이김 = 모든 line이 이김) = `unit × Σ(line별 odds 곱)`

Single/Multiple은 `lineCount == 1`로 같은 공식에서 떨어진다. **나눗셈이 없어 FLOOR 반올림이 단 한 번** — 돈 계산의 중간 반올림 누적이 원천 차단된다.

```java
long amount = BigDecimal.valueOf(unitStake.amount())
    .multiply(summedLineProducts)
    .setScale(0, RoundingMode.FLOOR)   // 단 한 번, 보수적(올림 아님)
    .longValueExact();
```

> **C/C++ 관점**: payout은 "보수적으로" FLOOR한다 — 북메이커가 사용자에게 1 minor unit이라도 *더* 주지 않게(올림이면 누적 손실). `longValueExact()`는 변환 중 정밀도 손실이 있으면 throw하므로 조용한 절단을 막는다.

### 조합 열거 + binomial

```java
static long binomial(int n, int k) {   // 곱셈 공식, n<=15에서 정확
  int kk = Math.min(k, n - k);
  long result = 1;
  for (int i = 0; i < kk; i++) result = result * (n - i) / (i + 1);
  return result;
}
```

- `combinations(n, k)`는 k-subset을 오름차순·사전순으로 열거하되 **pruning**(남은 원소가 k에 못 미치면 중단).
- `binomial`은 **곱셈 공식**(`result*(n-i)/(i+1)`)으로 — 팩토리얼을 직접 곱하면 금방 오버플로지만, 이 형태는 매 단계가 정수로 떨어져 n≤15에서 `long` 오버플로 없이 정확하다. `Math.min(k, n-k)`로 반복 횟수를 줄인다.

L4(max 15)가 폭발을 막는다 — 최악 C(15,7)=6435 line은 eager 열거로도 싸다.

## 결과

세 슬립 타입의 totalStake/maxPayout이 한 공식으로 계산됨. 014의 접수 흐름이 이걸 써서 지갑 차감액과 감사용 maxPayout을 만든다.

## 요약

- 슬립 = line 집합. Single/Multiple/System이 **한 공식**(totalStake = unit×lineCount, maxPayout = unit×Σproduct).
- 나눗셈 없음 → **FLOOR 반올림 단 한 번**(보수적).
- binomial 곱셈 공식 + pruning 조합 열거, L4가 C(15,7)=6435로 폭발 상한.

- **L1 (읽으면 됨)** — line 수 표(Single 1 / Multiple 1 / System C(N,K)).
