# 002 — Bet/BetLeg 불변식 테스트 (`test(domain): cover Bet/BetLeg invariants`)

## 개요 — 왜 이 변경이 필요한가

001의 aggregate 불변식은 영속 없이 순수 객체 수준에서 성립해야 한다. 영속(003)·정책(005) 전에 **aggregate 자체가 깨질 수 없음**을 단위 테스트로 박는다. 베팅 도메인은 가장 error-prone하므로 검증을 일찍·두껍게.

## 작업 순서

1. 슬립 타입별 구조 형태 단언.
2. K-of-N 파라미터 경계(상류 `BetSlipType`가 거름).
3. currency 불일치 거절.
4. STI 컬럼 왕복.
5. leg 순서/인덱스.
6. 생애주기 전이 + 불법 전이 throw.

## 작업 내역

영속을 띄우지 않는(no Testcontainers) 순수 단위 테스트라 빠르고 결정적이다.

- **구조 형태**: SINGLE = 1 leg, MULTIPLE ≥ 2, SYSTEM totalSelections = leg 수. 위반 시 `IllegalArgumentException`.
- **K-of-N 경계**: minWins ∈ 1..total, total ∈ 2..15는 `BetSlipType.System` 생성 시점(상류)에서 이미 거른다 — aggregate는 그 위를 신뢰.
- **currency**: stake KRW + maxPayout USD 같은 불일치 거절.
- **STI 왕복**: SYSTEM은 `slipType()` 복원 시 K-of-N 파라미터 유지, SINGLE/MULTIPLE은 파라미터 없음.
- **leg 순서**: 제출 순서대로 sequential index 부여.
- **전이**: PENDING→ACCEPTED/REJECTED 정상, 재수락·수락 후 거절은 throw.

## 결과

aggregate 불변식이 객체 수준에서 모두 보장됨을 박제. 이후 영속/정책 작업이 이 위에 안전하게 선다.

## 요약

- 구조·currency·STI 왕복·leg 순서·생애주기 전이를 단위로 검증(영속 없음).
- K-of-N 경계는 상류 타입이 책임지므로 여기선 신뢰만.
